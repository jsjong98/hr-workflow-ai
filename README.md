## ⚡ 빠른 시작 (더블클릭 3단계)

### 1단계 — Node.js 설치 (최초 1회만 · 5분)

👉 **https://nodejs.org/ko** 접속 → 초록색 **LTS 버전** 버튼 클릭 → 다운로드 → 설치
(설치 화면에서 모든 옵션 **기본값 그대로** Next → Next → Install)

---

### 2단계 — 소스코드 다운로드

이 GitHub 페이지에서 초록색 **Code** 버튼 클릭 → **Download ZIP** → 압축 해제

---

### 3단계 — 실행

🪟 **Windows 사용자** (대부분의 경우):
1. 압축 해제된 폴더 안에서 **`start.bat`** 파일을 **더블클릭**
2. "Windows가 PC를 보호했습니다" 경고창이 뜨면 → **"추가 정보"** 클릭 → **"실행"** 클릭
3. 검은 창(터미널)이 열리고 자동으로 설치 시작 (최초 1회, 약 2~3분)
4. 완료되면 **브라우저가 자동으로 열림** → `http://localhost:3000` 접속됨 🎉

> ⛔ **종료할 때**: 검은 창(터미널)을 **그냥 닫으면** 됩니다.
> 다음에 다시 쓸 때: `start.bat` 더블클릭 → 이번엔 설치 없이 바로 실행됨

🍎 **Mac 사용자**:
1. **`시작.command`** 파일을 **우클릭 → "열기" → "열기"** (처음 한 번만, 이후엔 더블클릭)

### ✅ 0단계: Node.js 설치 (필수 · 최초 1회)

> 반드시 먼저 설치하세요.

| 순서 | 설명 |
|------|------|
| **① 다운로드** | https://nodejs.org/ko 접속 → **LTS 버전** 다운로드 → 설치 (모든 옵션 기본값 OK) |
| **② 설치 확인** | 터미널(Mac) 또는 명령 프롬프트(Windows)를 **새로 열고** 아래 입력: |

node -v

- ✅ `v22.x.x` 같은 버전 번호가 나오면 → 설치 완료!
- ❌ `command not found` 또는 오류 → Node.js가 설치 안 된 것. ①번부터 다시

> 💡 **설치 직후 터미널/명령 프롬프트를 반드시 새로 열어야** node 명령어를 인식합니다.

### 1단계: 프로젝트 다운로드

#### 방법 A: Git Clone (권장)

아래 명령어를 **한 줄씩** 실행:

**첫 번째** — 프로젝트 다운로드:
```
git clone https://github.com/jsjong98/hr-workflow-ai.git
```

**두 번째** — 폴더로 이동:
```
cd hr-workflow-ai
```

#### 방법 B: ZIP 다운로드

1. GitHub 페이지 상단의 **Code** 버튼 → **Download ZIP** 클릭
2. ZIP 압축 해제

---

### 2단계: 폴더로 이동

> ⚠️ **이 단계를 건너뛰면 `npm install`이 실패합니다!**
> 반드시 `hr-workflow-ai` 폴더 **안에서** 명령어를 실행해야 합니다.

#### 🍎 Mac — 터미널 열기

`Spotlight (⌘ + Space)` → `터미널` 검색 → 열기

ZIP 다운로드 사용 시 아래 입력 (경로는 실제 압축 해제 위치에 맞게 수정):
```
cd ~/Downloads/hr-workflow-ai
```

#### 🪟 Windows — 명령 프롬프트(cmd) **관리자 권한**으로 열기

> ⚠️ **반드시 "명령 프롬프트"를 관리자 권한으로 실행하세요!**
> 일반 권한으로 열면 npm 설치 시 권한 오류가 발생합니다.
> PowerShell(파란색/보라색 창)에서는 npm이 차단될 수 있습니다.

**🔑 관리자 권한 명령 프롬프트 여는 법:**
1. **시작 메뉴**에서 **`cmd`** 검색
2. **"명령 프롬프트"** 가 나오면 → **우클릭** → **"관리자 권한으로 실행"** 클릭
3. **"이 앱이 디바이스를 변경할 수 있도록 허용하시겠어요?"** → **예** 클릭
4. 창 제목에 **"관리자: 명령 프롬프트"** 라고 나오면 성공 ✅

> 💡 **간편 방법:** `Win + R` → `cmd` 입력 → **`Ctrl + Shift + Enter`** (관리자 실행 단축키)

| ❌ 파란색/보라색 창 | ❌ 일반 검은색 창 | ✅ 관리자 검은색 창 |
|------|------|------|
| PowerShell (npm 차단됨) | 권한 부족 오류 가능 | **"관리자: 명령 프롬프트"** ← 이것을 사용! |

ZIP 다운로드 사용 시 아래 입력:
```
cd %USERPROFILE%\Downloads\hr-workflow-ai
```

> 💡 압축 해제 경로가 다르면 `cd` 뒤의 경로를 실제 폴더 위치로 변경하세요.

#### ✅ 올바른 폴더에 있는지 확인

아래 명령어를 입력해서 `package.json`이 보이면 정상입니다:

Mac:
```
ls package.json
```

Windows:
```
dir package.json
```

- ✅ `package.json` 파일이 보이면 → 올바른 위치!
- ❌ `No such file` 또는 `파일을 찾을 수 없습니다` → 폴더 경로가 잘못된 것. `cd` 명령어 확인

### 3단계: 패키지 설치 (최초 1회)

> 💡 이 명령어는 **최초 1회만** 실행하면 됩니다. 다음부터는 4단계만 실행하세요.

## ❓ 자주 발생하는 문제

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| `npm: command not found` | Node.js 미설치 | 0단계 Node.js 설치 후 **터미널을 새로 열기** |
| `npm ERR! enoent` / `no such file` | 잘못된 폴더에서 실행 | `cd hr-workflow-ai` 폴더로 이동 후 `ls package.json` (Mac) 또는 `dir package.json` (Windows)으로 확인 |
| 여러 줄 붙여넣기 시 오류 | 명령어를 한꺼번에 복사함 | **반드시 한 줄씩** 복사 → 붙여넣기 → Enter → 완료 확인 → 다음 줄 |
| PowerShell에서 npm 차단 | Windows PowerShell 보안 정책 | **관리자 권한 명령 프롬프트** 사용 (시작 → `cmd` 검색 → 우클릭 → 관리자 권한으로 실행) |
| `EACCES` / `EPERM` 권한 오류 | 관리자 권한 없이 실행 | **Windows:** 관리자 권한 cmd 사용 / **Mac:** `sudo npm install --legacy-peer-deps` |
| 포트 3000 사용 중 | 이미 실행 중인 서버 | 기존 터미널에서 `Ctrl + C`로 종료 후 다시 `npm run dev` |

## 🔑 AI 기능 사용 (선택사항)

AI 워크플로우 생성 기능을 사용하려면 **OpenAI API Key**가 필요합니다.

프로젝트 폴더에 `.env.local` 파일을 생성하고 아래 내용을 입력:

```
OPENAI_API_KEY=sk-xxxx...
```

> API Key 없이도 CSV 업로드, 노드 편집, PPT/Excel 내보내기 등 대부분의 기능을 사용할 수 있습니다.

## 📖 주요 기능

| 기능 | 설명 |
|------|------|
| **CSV 업로드** | L2→L3→L4→L5 계층 CSV 파일 드래그&드롭 또는 클릭 업로드 |
| **팔레트 탐색** | L2→L3→L4→L5 계층 아코디언 탐색, 클릭/드래그로 캔버스 배치 |
| **새 데이터 추가** | 팔레트 하단 입력 폼에서 L5 항목 수동 추가 (이름·설명·수행주체 입력) |
| **캔버스 편집** | 노드 드래그&드롭, 화살표 연결, 양방향 화살표 전환 |
| **수영레인(Swimlane)** | 임원·팀장·HR담당자·구성원 4분할 레인 시트 |
| **노드 메타데이터 편집** | 노드 더블클릭 → 수행주체·I/O·사용시스템·관리주체 등 상세 정보 입력 |
| **사용 시스템 태그** | L5 노드 하단에 HR시스템·그룹웨어·오피스·수작업·기타툴 태그 자동 표시 |
| **다중 시트** | 시트 탭 추가·삭제·복제·이름변경 |
| **AI Workflow** | 🤖 버튼 → AI가 자동으로 워크플로우 생성 |
| **💾 JSON 저장/불러오기** | 전체 시트(노드·엣지·레이아웃) JSON으로 저장 후 재편집 가능 |
| **📊 PPT 내보내기** | 현재 시트를 다이어그램·순서·연결목록·노드상세 포함 PPT로 저장 |
| **📋 전체 PPT 내보내기** | 모든 시트를 1장씩 다이어그램 슬라이드로 묶어 저장 (사용 시스템 태그 포함) |
| **📗 통합 Excel** | 원본 CSV + 캔버스 수정/추가 내용 통합 xlsx — 변경행 노란색, 신규행 초록색으로 강조 |
| **📋 전체 시트 Excel** | 모든 시트를 각각 별도 워크시트로 분리하여 xlsx 1개로 내보내기 |

## 📤 Excel 내보내기 상세

### 📗 통합 Excel (원본 CSV 있을 때)

원본 CSV 데이터와 캔버스 편집 내용을 비교·병합하여 색상으로 구분합니다.

| 색상 | 의미 |
|------|------|
| 흰색 | 변경 없음 (원본 그대로) |
| 🟡 노란색 | 캔버스에서 수정된 행 |
| 🟢 초록색 | 팔레트에서 새로 추가된 행 |

**삽입 & 재번호 동작:**
새 항목을 팔레트에서 추가하고 ID를 기존 번호(예: `1.1.1.1`)로 지정하면, 해당 L4 아래 전체 L5 항목이 팔레트 순서대로 자동 재번호됩니다.

예) `1.1.1.1`에 "Test" 삽입 시:
- `1.1.1.1` = Test (초록, 신규)
- 기존 `1.1.1.1` → `1.1.1.2` (노란색, ID 변경)
- 기존 `1.1.1.2` → `1.1.1.3`, … 순으로 밀림

### 📋 전체 시트 Excel (캔버스 기반)

원본 CSV와 무관하게 현재 캔버스에 올라온 노드만 내보냅니다.
시트 탭 이름 그대로 각 워크시트로 분리되어 xlsx 1개 파일로 저장됩니다.
